# FAQ知识库管理系统功能模块说明

## 1. 配置模块 (config.py)

### 功能描述
- 定义系统全局配置参数，包括API密钥、文件路径、向量维度等
- 提供统一的配置管理，方便系统各模块使用

### 核心配置项
| 配置项 | 类型 | 描述 | 默认值 |
|--------|------|------|--------|
| `DASHSCOPE_API_KEY` | str | DashScope API密钥 | 环境变量或空字符串 |
| `DASHSCOPE_EMBEDDING_MODEL` | str | 嵌入模型名称（已弃用） | "text-embedding-v1" |
| `FAISS_INDEX_PATH` | str | FAISS索引文件路径 | "./data/faiss_index" |
| `VECTOR_DIMENSION` | int | 向量维度 | 1024 |
| `FAQ_FILE_PATH` | str | FAQ文件路径 | "./data/faq.json" |
| `TOP_K` | int | 检索结果数量 | 3 |

### 使用场景
- 系统初始化时加载配置
- 各模块通过`settings`对象访问配置参数

## 2. 数据加载模块 (data_loader.py)

### 功能描述
- 解析FAQ文件，创建文档对象
- 构建和管理向量索引
- 提供索引的保存和加载功能

### 核心类和方法

#### `FAQDataLoader` 类
| 方法 | 描述 | 参数 | 返回值 |
|------|------|------|--------|
| `parse_faq_file` | 解析FAQ文件 | `file_path`: str | `List[Dict]` |
| `create_documents` | 创建文档对象 | `faq_items`: List[Dict] | `List[Document]` |
| `build_vector_index` | 构建向量索引 | `documents`: List[Document] | `VectorStoreIndex` |
| `save_index` | 保存索引 | `index`: VectorStoreIndex, `file_path`: str | None |
| `load_index` | 加载索引 | 无 | `VectorStoreIndex` |
| `initialize_faq_system` | 初始化FAQ系统 | `force_rebuild`: bool | `VectorStoreIndex` |

### 使用场景
- 系统启动时初始化FAQ系统
- 知识库更新后重建索引
- 查询时加载索引进行检索

## 3. 知识库管理模块 (kb_manager.py)

### 功能描述
- 管理FAQ条目的增删改查
- 支持批量导入导出FAQ
- 提供知识库版本管理（备份、恢复、删除）
- 重建向量索引

### 核心类和方法

#### `KnowledgeBaseManager` 类
| 方法 | 描述 | 参数 | 返回值 |
|------|------|------|--------|
| `get_all_faqs` | 获取所有FAQ | 无 | `List[Dict]` |
| `get_faq_by_id` | 根据ID获取FAQ | `faq_id`: int | `Dict` |
| `add_faq` | 添加FAQ | `question`: str, `answer`: str, `auto_rebuild`: bool | `Dict` |
| `update_faq` | 更新FAQ | `faq_id`: int, `question`: str, `answer`: str, `auto_rebuild`: bool | `bool` |
| `delete_faq` | 删除FAQ | `faq_id`: int, `auto_rebuild`: bool | `bool` |
| `search_faqs` | 搜索FAQ | `keyword`: str | `List[Dict]` |
| `update_knowledge_base` | 更新知识库 | `new_faqs`: List[Dict], `merge_strategy`: str | `bool` |
| `export_knowledge_base` | 导出知识库 | `file_path`: str, `format_type`: str | `bool` |
| `import_knowledge_base` | 导入知识库 | `file_path`: str, `merge_strategy`: str | `bool` |
| `backup_knowledge_base` | 备份知识库 | 无 | `str` |
| `restore_knowledge_base` | 恢复知识库 | `backup_file`: str | `bool` |
| `delete_backup` | 删除备份 | `backup_file`: str | `bool` |
| `get_backup_list` | 获取备份列表 | 无 | `List[Dict]` |
| `get_statistics` | 获取统计信息 | 无 | `Dict` |
| `validate_faq` | 验证FAQ格式 | `faq`: Dict | `bool` |
| `rebuild_index` | 重建索引 | `force`: bool | `bool` |

### 使用场景
- 管理FAQ条目
- 批量导入导出知识库
- 备份和恢复知识库
- 重建向量索引

## 4. Web API模块 (webapi.py)

### 功能描述
- 提供RESTful API接口
- 处理用户的问答请求
- 管理知识库
- 系统管理

### API接口

#### 1. 问答接口
| 接口 | 方法 | 描述 | 请求体 | 响应体 |
|------|------|------|--------|--------|
| `/api/v1/chat` | POST | 发送问题获取答案 | `{"question": "...", "session_id": "..."}` | `{"answer": "...", "session_id": "...", "sources": [...], "timestamp": "..."}` |
| `/api/v1/chat/history` | GET | 获取对话历史 | `session_id` (query) | `{"session_id": "...", "history": [...], "total_count": 10}` |

#### 2. 知识库管理接口
| 接口 | 方法 | 描述 | 请求体 | 响应体 |
|------|------|------|--------|--------|
| `/api/v1/knowledge/faq` | POST | 添加FAQ条目 | `{"question": "...", "answer": "..."}` | `{"id": 1, "question": "...", "answer": "..."}` |
| `/api/v1/knowledge/faq/{faq_id}` | PUT | 更新FAQ条目 | `{"question": "...", "answer": "..."}` | `{"id": 1, "question": "...", "answer": "..."}` |
| `/api/v1/knowledge/faq/{faq_id}` | DELETE | 删除FAQ条目 | N/A | `{"message": "FAQ条目 1 已成功删除"}` |
| `/api/v1/knowledge/faq` | GET | 查询FAQ列表 | `keyword` (query), `limit` (query), `offset` (query) | `[{"id": 1, "question": "...", "answer": "..."}]` |
| `/api/v1/knowledge/batch` | POST | 批量导入FAQ | `{"faqs": [...], "merge_strategy": "append"}` | `{"message": "成功导入 10 条FAQ", "merge_strategy": "append", "imported_count": 10}` |

#### 3. 系统管理接口
| 接口 | 方法 | 描述 | 请求体 | 响应体 |
|------|------|------|--------|--------|
| `/api/v1/system/health` | GET | 系统健康检查 | N/A | `{"status": "healthy", "timestamp": "...", "components": {...}}` |
| `/api/v1/system/rebuild-index` | POST | 重建向量索引 | `force` (query) | `{"message": "向量索引重建成功", "timestamp": "...", "force": true}` |

### 使用场景
- 客户端通过API调用系统功能
- 第三方系统集成
- 系统监控和管理

## 5. 查询脚本模块 (ask.py)

### 功能描述
- 从磁盘加载索引到内存

- 检查索引文件是否存在
- 加载索引到内存
- 执行查询
- 输出查询结果和相关FAQ条目

### 使用场景
- 快速测试系统查询功能
- 命令行方式使用系统
- 调试和验证索引质量

### 技术特点
- **高效向量检索**：使用FAISS实现高性能相似度检索
- **灵活的知识库管理**：支持增删改查、批量导入导出、版本管理

### 应用场景
- **客户服务**：自动回答常见问题
- **内部知识库**：企业内部知识管理
- **智能助手**：集成到聊天机器人中
- **信息检索**：快速查询文档信息

## 8. 系统限制与注意事项

### 限制
- **会话存储**：当前使用内存存储，重启服务后会话历史会丢失（生产环境建议使用Redis）
- **向量维度**：固定为1024，与使用的嵌入模型相关
- **API密钥**：需要配置DashScope API密钥才能使用嵌入模型

### 注意事项
- **索引重建**：知识库更新后需要重建索引才能生效
- **数据备份**：定期备份知识库，防止数据丢失
- **性能优化**：对于大量FAQ，可考虑使用更高级的向量数据库
- **安全性**：API密钥应妥善保管，避免泄露
