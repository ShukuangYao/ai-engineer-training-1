# 混合RAG系统功能模块说明

## 1. 配置模块

### 功能说明
配置模块负责管理系统的全局参数，包括检索配置、权重配置、阈值设置等。

### 核心参数
- **RETRIEVAL_TOP_K**：检索返回的top-k结果数，默认为5
- **RETRIEVAL_VECTOR_WEIGHT**：向量检索权重，默认为0.5
- **RETRIEVAL_KEYWORD_WEIGHT**：关键词检索权重，默认为0.3
- **RETRIEVAL_GRAPH_WEIGHT**：图谱检索权重，默认为0.2
- **ERROR_PROPAGATION_THRESHOLD**：错误传播阈值，默认为0.3

### 配置文件
- **config.py**：定义全局配置参数，支持从环境变量读取API密钥

## 2. 数据模型模块

### 功能说明
数据模型模块定义了系统中使用的核心数据结构，包括实体、关系、文档、检索结果和图谱结果等。

### 核心数据类
- **Entity**：实体类，表示知识图谱中的实体，包含名称、类型和置信度
- **Relationship**：关系类，表示知识图谱中实体之间的关系，包含源实体、目标实体、关系类型和置信度
- **Document**：文档类，表示检索系统中的文档，包含ID、内容、元数据和嵌入向量
- **RetrievalResult**：检索结果类，表示检索系统返回的结果，包含文档、分数和检索来源
- **GraphResult**：图谱结果类，表示知识图谱推理的结果，包含实体列表、关系列表、置信度和推理路径

### 数据模型文件
- **models.py**：定义所有数据模型类

## 3. 嵌入服务模块

### 功能说明
嵌入服务模块负责将文本转换为向量表示，支持使用通义千问API和本地模型降级。

### 核心功能
- **文本向量化**：将文本编码为向量表示
- **API调用**：调用通义千问text-embedding-v3 API
- **降级机制**：当API调用失败时，自动降级到本地SentenceTransformer模型

### 实现类
- **QwenEmbedding**：通义千问文本嵌入服务，支持中文理解

### 嵌入服务文件
- **embedding.py**：实现文本嵌入服务

## 4. 混合检索模块

### 功能说明
混合检索模块整合了向量检索、关键词检索和图谱推理三种检索方式，提供多源检索能力。

### 核心功能
- **向量检索**：基于文本嵌入的相似度搜索
- **关键词检索**：基于jieba分词的关键词匹配
- **图谱推理**：基于Neo4j图数据库的多跳关系推理
- **联合评分**：结合三个检索模块的结果，计算综合得分

### 实现方法
- **vector_search**：执行向量检索，返回相似度排序结果
- **keyword_search**：执行关键词检索，返回匹配度排序结果
- **graph_reasoning**：执行图谱推理，返回多跳关系结果
- **calculate_joint_score**：计算联合评分，整合多源检索结果

### 检索模块文件
- **retrieval.py**：实现混合检索逻辑

## 5. 多跳问答模块

### 功能说明
多跳问答模块是系统的核心功能，实现了多跳推理和问答能力。

### 核心功能
- **实体提取**：从用户问题中提取相关实体
- **多源检索**：并行执行向量检索、关键词检索和图谱推理
- **联合评分**：结合多源检索结果，计算综合得分
- **错误传播防护**：检测并过滤低质量结果，防止错误传播
- **答案生成**：基于检索结果和图谱推理生成最终答案

### 实现方法
- **extract_entities_from_query**：从查询中提取实体
- **multi_hop_qa**：执行多跳问答流程
- **generate_final_answer**：生成最终答案

### 多跳问答文件
- **hybrid_rag_system.py**：实现混合RAG系统的核心功能
- **Allin1_hybrid_rag.py**：综合实现混合RAG系统的完整功能

## 10. 系统特点

### 技术特点
- **多源检索**：融合向量、关键词和图谱三种检索方式
- **多跳推理**：支持多跳关系查询，解决复杂问题
- **错误防护**：通过置信度检测和过滤机制，防止错误信息传播
- **中文优化**：使用通义千问和jieba分词，优化中文理解和处理
- **模块化设计**：各模块职责清晰，易于维护和扩展
- **降级机制**：当API调用失败时，自动降级到本地模型

### 应用场景
- **企业知识库问答**：基于企业文档和知识图谱回答问题
- **金融分析**：分析企业之间的控股关系和关联网络
- **信息检索**：快速检索和整合多源信息
- **智能助手**：提供基于知识的智能问答服务

## 11. 系统配置与部署

### 环境要求
- **Python 3.8+**：系统运行环境
- **Neo4j**：图数据库，用于存储实体关系
- **通义千问API密钥**：用于文本嵌入和大语言模型服务
- **依赖库**：dashscope、neo4j、sentence-transformers、jieba、scikit-learn等

### 配置步骤
1. **安装依赖**：`pip install -r requirements.txt`
2. **设置API密钥**：设置环境变量 `DASHSCOPE_API_KEY`
3. **启动Neo4j**：确保Neo4j数据库服务运行
4. **准备数据**：准备文本数据文件（如company.txt）
5. **运行演示**：执行 `python Allin1_hybrid_rag.py`

### 性能优化
- **批量处理**：批量处理文档，减少API调用次数
- **缓存机制**：缓存嵌入向量，避免重复计算
- **并行检索**：并行执行多源检索，提高响应速度
- **索引优化**：优化关键词索引，提高检索效率
