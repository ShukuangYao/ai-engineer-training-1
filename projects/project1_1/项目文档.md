## 📖 项目概述

### 项目简介

多任务问答助手是一个基于 LangChain 框架构建的智能对话系统，能够理解用户意图并自动调用相应的工具来完成任务。系统支持自然语言对话、天气查询和信息搜索等多种功能。

### 核心特性

- ✅ **智能意图识别**: 自动识别用户意图，决定是否需要调用工具
- ✅ **工具自动调用**: 基于 Function Calling 机制，LLM 自动决定工具调用
- ✅ **对话历史管理**: 维护上下文，支持多轮对话
- ✅ **错误处理**: 完善的异常处理和用户友好的错误提示
- ✅ **日志记录**: 结构化日志系统，便于调试和监控
- ✅ **配置管理**: 统一的配置管理，支持环境变量

### 技术栈

| 技术 | 版本 | 用途 |
|------|------|------|
| Python | 3.11+ | 编程语言 |
| LangChain | 0.1.17 | 对话链构建 |
| DeepSeek API | - | 大语言模型 |

| Loguru | 0.7.2 | 日志记录 |

## 🏗️ 技术架构

### 架构层次

```
┌─────────────────────────────────────────┐
│         用户交互层 (Presentation)        │

└─────────────────────────────────────────┘
                    ↓
┌─────────────────────────────────────────┐
│         业务逻辑层 (Business Logic)      │
│  - QAAgent: 对话管理                    │
│  - 意图识别                             │
│  - 工具调用决策                         │
│  - 回复生成                             │
└─────────────────────────────────────────┘
                    ↓
┌─────────────────────────────────────────┐
│         服务层 (Service Layer)          │
│  - 天气查询服务                         │
│  - 信息搜索服务                         │

└─────────────────────────────────────────┘
                    ↓
┌─────────────────────────────────────────┐
│         数据访问层 (Data Access)        │

│  - 配置读取                             │
│  - 日志写入                             │
└─────────────────────────────────────────┘
```

### 设计模式

1. **代理模式 (Proxy Pattern)**
   - `QAAgent` 作为用户和工具之间的代理
   - 统一管理工具调用和结果处理

2. **策略模式 (Strategy Pattern)**
   - 不同工具使用不同的执行策略
   - 便于扩展新工具

3. **单例模式 (Singleton Pattern)**
   - `Settings` 类使用单例模式
   - 确保配置的一致性

4. **工厂模式 (Factory Pattern)**
   - `create_qa_agent()` 函数创建代理实例
   - 封装复杂的初始化逻辑

## 🔧 核心模块详解

### 1. QAAgent (问答代理)

**位置**: `agents/qa_agent.py`

**职责**:
- 管理对话历史和上下文
- 处理用户输入
- 调用工具（如果需要）
- 生成回复

**关键方法**:

```python
def chat(user_input: str) -> Dict[str, Any]:
    """
    处理用户输入并生成回复

    Args:
        user_input: 用户输入文本

    Returns:
        {
            'response': str,              # 助手回复
            'tools_used': List[str],      # 使用的工具列表
            'processing_time_ms': float   # 处理时间（毫秒）
        }
    """
```

**工作流程**:
1. 接收用户输入
2. 更新对话历史
3. 调用 LLM（带工具绑定）
4. 如果 LLM 返回工具调用，执行工具
5. 将工具结果传给 LLM 生成最终回复
6. 保存对话历史
7. 返回结果

### 2. 工具模块

#### AmapWeatherTool (天气查询工具)

**位置**: `tools/amap_weather_tool.py`

**功能**: 通过高德地图 API 查询城市天气

**API 端点**: `https://restapi.amap.com/v3/weather/weatherInfo`

**参数**:
- `city_name`: 城市名称（如 "北京"）

**返回格式**:
```python
{
    'success': bool,
    'data': str,      # 格式化的天气信息
    'error': str      # 错误信息（如果有）
}
```

#### TavilySearchTool (信息搜索工具)

**位置**: `tools/tavily_search_tool.py`

**功能**: 通过 Tavily Search API 搜索网络信息

**API 端点**: `https://api.tavily.com/search`

**参数**:
- `query`: 搜索关键词
- `max_results`: 最大结果数量（默认 5）

**返回格式**:
```python
{
    'success': bool,
    'data': {
        'query': str,
        'answer': str,           # AI 生成的总结
        'results': List[Dict],   # 搜索结果列表
        'total_results': int
    },
    'error': str
}
```

## 🔌 API 接口说明

### 内部 API

#### QAAgent.chat()

**方法签名**:
```python
def chat(user_input: str) -> Dict[str, Any]
```

**参数**:
- `user_input` (str): 用户输入文本

**返回值**:
```python
{
    'response': str,              # 助手回复文本
    'tools_used': List[str],      # 使用的工具列表，如 ['weather_query']
    'processing_time_ms': float   # 处理时间（毫秒）
}
```

**示例**:
```python
agent = create_qa_agent()
result = agent.chat("查询北京天气")
print(result['response'])
# 输出: "根据高德地图API，北京当前天气情况如下：..."
```

## ⚠️ 错误处理

### 错误类型

1. **配置错误**: API 密钥未配置或无效
2. **API 错误**: 外部 API 调用失败
3. **网络错误**: 网络连接问题
4. **工具错误**: 工具执行失败

### 错误处理策略

```python
try:
    result = tool.execute(params)
except Exception as e:
    # 记录错误日志
    app_logger.error(f"工具执行失败: {e}")
    # 返回友好的错误信息
    return {
        'success': False,
        'error': f"操作失败: {str(e)}"
    }
```

### 用户友好的错误提示

系统会提供清晰的错误信息和解决建议：

```
❌ Tavily API 请求失败 (432 错误)。可能的原因：
1. API 密钥无效或已过期
2. API 配额已用完
3. 请求参数格式不正确
```

## 🚀 性能优化

### 优化策略

1. **对话历史限制**: 限制历史记录数量，避免上下文过长
2. **工具结果缓存**: 可以集成 Redis 缓存工具调用结果
3. **异步处理**: 可以改为异步处理提高并发性能
4. **连接池**: 使用 HTTP 连接池复用连接

### 性能指标

- **平均响应时间**: < 3秒（包含工具调用）
- **普通对话响应时间**: < 1秒
- **工具调用时间**: 1-2秒（取决于外部 API）

## ❓ 常见问题

### Q1: 如何添加新工具？

**A**:
1. 在 `tools/` 目录创建工具类
2. 在 `tools/tool_schemas.py` 定义参数模式
3. 在 `agents/qa_agent.py` 的 `_create_tools()` 方法中注册工具

### Q2: 如何切换 LLM 提供商？

**A**: 修改 `config/settings.py` 中的 API 配置，或修改环境变量 `DEEPSEEK_API_KEY` 和 `DEEPSEEK_API_BASE`。

### Q3: 如何处理 API 配额限制？

**A**:
1. 实现结果缓存
2. 添加请求限流
3. 使用多个 API 密钥轮换

### Q4: 如何持久化对话历史？

**A**: 在 `QAAgent` 类中添加数据库存储逻辑，在 `chat()` 方法中保存对话记录。

### Q5: 如何调试工具调用问题？

**A**:
1. 查看日志文件 `logs/app_*.log`
2. 启用 DEBUG 日志级别
3. 检查工具返回的数据格式
