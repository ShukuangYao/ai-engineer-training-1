## 2. 数据流图

### 2.1 订单查询数据流

```mermaid
sequenceDiagram
    participant User as 用户
    participant Main as main.py
    participant UserProxy as 用户代理
    participant GroupChat as 群组聊天
    participant OrderAgent as 订单查询专员
    participant OrderTool as 订单查询工具
    participant APIClient as API客户端
    participant Retry as 重试机制
    participant FastAPI as FastAPI服务器
    participant DeepSeek as DeepSeek API

    User->>Main: 查询："我的订单ORD001为什么还没发货？"
    Main->>UserProxy: 启动对话
    UserProxy->>GroupChat: 发送查询消息
    GroupChat->>OrderAgent: 分配任务给订单查询专员
    OrderAgent->>DeepSeek: 分析查询，提取订单ID
    DeepSeek-->>OrderAgent: 返回分析结果（订单ID: ORD001）
    OrderAgent->>OrderTool: 调用 get_order_info("ORD001")
    OrderTool->>APIClient: 请求订单信息
    APIClient->>Retry: 发起HTTP请求（带重试）
    Retry->>FastAPI: GET /api/orders/ORD001
    FastAPI-->>Retry: 返回订单数据（JSON）
    Retry-->>APIClient: 返回响应
    APIClient-->>OrderTool: 返回订单信息
    OrderTool-->>OrderAgent: 返回格式化结果
    OrderAgent->>DeepSeek: 生成回复（基于订单信息）
    DeepSeek-->>OrderAgent: 返回回复文本
    OrderAgent->>GroupChat: 发送回复
    GroupChat->>UserProxy: 转发回复
    UserProxy->>Main: 返回最终结果
    Main->>User: 显示回复
```

## 3. 智能体交互流程图

### 3.1 智能体协作流程

```mermaid
flowchart TD
    Start([用户发起查询]) --> ParseQuery{解析查询内容}
    ParseQuery -->|包含订单ID| ExtractID[提取订单ID]
    ParseQuery -->|不包含订单ID| DefaultID[使用默认订单ID]

    ExtractID --> InitAgents[初始化智能体团队]
    DefaultID --> InitAgents

    InitAgents --> CustomerService[客服接待员<br/>初步分析问题]
    CustomerService --> Classify{问题分类}

    Classify -->|订单问题| OrderQuery[订单查询专员<br/>查询订单信息]
    Classify -->|物流问题| Logistics[物流跟踪专员<br/>查询物流信息]
    Classify -->|综合问题| Both[同时查询订单和物流]

    OrderQuery --> OrderAPI[调用订单API]
    Logistics --> LogisticsAPI[调用物流API]
    Both --> OrderAPI
    Both --> LogisticsAPI

    OrderAPI --> RetryCheck1{请求成功?}
    RetryCheck1 -->|失败| Retry1[指数退避重试]
    Retry1 --> OrderAPI
    RetryCheck1 -->|成功| OrderResult[获取订单信息]

    LogisticsAPI --> RetryCheck2{请求成功?}
    RetryCheck2 -->|失败| Retry2[指数退避重试]
    Retry2 --> LogisticsAPI
    RetryCheck2 -->|成功| LogisticsResult[获取物流信息]

    OrderResult --> Summary[客服主管<br/>汇总结果]
    LogisticsResult --> Summary

    Summary --> GenerateReply[生成完整回复]
    GenerateReply --> Display[显示回复给用户]
    Display --> End([结束])
```

### 3.2 重试机制流程图

```mermaid
flowchart TD
    Start([发起API请求]) --> Request[发送HTTP请求]
    Request --> CheckStatus{检查响应状态}

    CheckStatus -->|200 OK| Success[请求成功<br/>返回数据]
    CheckStatus -->|404 Not Found| NoRetry[不重试<br/>返回错误]
    CheckStatus -->|500/503 错误| RetryCheck{检查重试次数}
    CheckStatus -->|网络错误| RetryCheck

    RetryCheck -->|未达到最大次数| CalculateWait[计算等待时间<br/>指数退避]
    RetryCheck -->|达到最大次数| MaxRetries[达到最大重试次数<br/>抛出异常]

    CalculateWait --> Wait[等待指定时间]
    Wait --> IncrementAttempt[增加重试计数]
    IncrementAttempt --> Request

    Success --> End([结束])
    NoRetry --> End
    MaxRetries --> End
```

## 5. 状态图

### 5.1 智能体状态转换

```mermaid
stateDiagram-v2
    [*] --> Idle: 初始化
    Idle --> Receiving: 接收消息
    Receiving --> Analyzing: 分析消息
    Analyzing --> Thinking: 思考处理方案
    Thinking --> Executing: 执行操作
    Executing --> Waiting: 等待API响应
    Waiting --> Processing: 处理响应
    Processing --> Generating: 生成回复
    Generating --> Sending: 发送回复
    Sending --> Idle: 完成
    Sending --> Error: 发生错误
    Error --> Retrying: 重试
    Retrying --> Executing: 重新执行
    Retrying --> [*]: 达到最大重试次数
```

### 5.2 API请求状态转换

```mermaid
stateDiagram-v2
    [*] --> Pending: 发起请求
    Pending --> Sending: 发送HTTP请求
    Sending --> Success: 200 OK
    Sending --> NotFound: 404 Not Found
    Sending --> ServerError: 500/503 错误
    Sending --> NetworkError: 网络错误
    ServerError --> Retrying: 重试
    NetworkError --> Retrying: 重试
    Retrying --> Sending: 重新发送
    Retrying --> Failed: 达到最大重试次数
    Success --> [*]: 返回数据
    NotFound --> [*]: 返回错误
    Failed --> [*]: 抛出异常
```

## 7. 错误处理流程图

### 7.1 错误处理流程

```mermaid
flowchart TD
    Start([执行操作]) --> Try[尝试执行]
    Try --> CheckError{发生错误?}

    CheckError -->|无错误| Success[操作成功]
    CheckError -->|有错误| ErrorType{错误类型}

    ErrorType -->|404 Not Found| NoRetry[不重试<br/>返回友好错误信息]
    ErrorType -->|500/503 错误| RetryCheck{检查重试次数}
    ErrorType -->|网络错误| RetryCheck
    ErrorType -->|其他错误| LogError[记录错误日志]

    RetryCheck -->|未达到最大次数| CalculateWait[计算等待时间]
    RetryCheck -->|达到最大次数| MaxRetries[达到最大重试次数]

    CalculateWait --> Wait[等待指定时间]
    Wait --> IncrementAttempt[增加重试计数]
    IncrementAttempt --> Try

    MaxRetries --> LogError
    LogError --> UserFriendly[生成用户友好错误信息]
    UserFriendly --> ReturnError[返回错误信息]

    Success --> End([结束])
    NoRetry --> End
    ReturnError --> End
```

## 9. 日志记录流程图

### 9.1 日志记录流程

```mermaid
flowchart TD
    Start([发生事件]) --> CheckLevel{检查日志级别}

    CheckLevel -->|DEBUG| DebugLog[记录DEBUG日志]
    CheckLevel -->|INFO| InfoLog[记录INFO日志]
    CheckLevel -->|WARNING| WarningLog[记录WARNING日志]
    CheckLevel -->|ERROR| ErrorLog[记录ERROR日志]

    DebugLog --> FormatLog[格式化日志消息]
    InfoLog --> FormatLog
    WarningLog --> FormatLog
    ErrorLog --> FormatLog

    FormatLog --> CheckConsole{输出到控制台?}
    CheckConsole -->|是| ConsoleOutput[输出到控制台<br/>Rich格式化]
    CheckConsole -->|否| SkipConsole[跳过控制台输出]

    ConsoleOutput --> CheckFile{输出到文件?}
    SkipConsole --> CheckFile

    CheckFile -->|是| FileOutput[写入日志文件<br/>UTF-8编码]
    CheckFile -->|否| SkipFile[跳过文件输出]

    FileOutput --> RotateCheck{文件大小检查}
    RotateCheck -->|超过限制| RotateFile[轮转日志文件]
    RotateCheck -->|未超过| KeepFile[保持当前文件]

    RotateFile --> End([日志记录完成])
    KeepFile --> End
```
