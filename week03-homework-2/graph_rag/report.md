# 融合文档检索与图谱推理的多跳问答系统

## 实验环境

- **操作系统**: macOS 14.0
- **Python 版本**: 3.9.13
- **Neo4j 版本**: 5.15.0
- **CPU**: Apple M1 Pro
- **内存**: 16GB
- **索引框架**: llama-index 0.8.39
- **大语言模型**: qwen-plus (通过 DashScope API)

## 系统架构

### 核心组件

1. **文档检索模块 (RAG)**
   - 使用 llama-index 构建向量索引（可用时）
   - 支持语义搜索和文档相似度计算
   - 实现关键词匹配回退机制
   - 自动判断信息来源类型

2. **知识图谱模块 (KG)**
   - 使用 Neo4j 构建公司控股关系图谱
   - 支持多跳推理和路径查找
   - 实现层级查询功能

3. **联合评分机制**
   - 公式：`combined_score = α * rag_score + (1 - α) * kg_score`
   - 默认 α = 0.5，可自定义权重
   - 自动判断信息来源（文档检索/图谱推理/混合）
   - 动态权重调整：当一方置信度较低时自动降低其权重

4. **一致性校验**
   - 关系一致性检查：防止重复和矛盾关系
   - 循环关系检测：避免控股关系形成闭环
   - 实体验证：确保关系两端实体都存在

5. **可解释性输出**
   - 显示推理路径和控股层级
   - 展示 RAG 和 KG 各自的评分
   - 说明信息来源类型

## 功能实现

### 1. 知识图谱构建

#### 实体提取
- 从财报文本中提取公司实体
- 支持提取公司属性（如持股比例）
- 使用 LLM 进行结构化提取

#### 关系提取
- 识别公司间的控股关系
- 提取持股比例等关系属性
- 验证关系模式的有效性

#### 一致性校验
```python
# 关系一致性检查
check_relationship_consistency(source, target, rel_type)

# 循环关系检测
check_cyclic_relationship(source, target)
```

### 2. 文档检索

使用 llama-index 实现：
- 文档向量化和索引构建
- 语义相似度搜索
- Top-K 相关文档检索
- 文档评分和聚合
- 关键词匹配回退机制（当 LlamaIndex 不可用时）

**关键实现**：
```python
class DocumentRetriever:
    def __init__(self, documents: List[str]):
        self.documents = documents
        self.index = None
        self.query_engine = None
        self.use_fallback = not LLAMA_INDEX_AVAILABLE

    def retrieve_with_source(self, query: str, top_k: int = 3):
        if not self.use_fallback and self.index is not None:
            return self._llama_retrieve(query, top_k)
        else:
            return self._fallback_retrieve(query, top_k)
```

### 3. 多跳问答

#### 查询流程
1. **问题分析**：提取查询中的关键词和实体
2. **双路检索**：
   - 文档检索：查找相关财报文档
   - 图谱推理：查找控股关系路径
3. **评分融合**：使用 α 参数融合两个评分
4. **答案生成**：基于融合结果生成最终答案

#### Cypher 查询示例
```cypher
// 多跳控股关系查询
MATCH path = (parent:Company {name: $parent_name})-[:CONTROLS*1..]->(subsidiary:Company)
RETURN subsidiary.name as subsidiary,
       length(path) as depth,
       [node in nodes(path) | node.name] as path_nodes
ORDER BY depth, subsidiary.name
```

## 系统优缺点分析

### 优点

1. **双路检索融合**：结合文档检索和图谱推理的优势
2. **灵活的评分机制**：支持自定义 α 参数调整权重
3. **完善的一致性校验**：防止错误传播和数据污染
4. **清晰的可解释性**：展示推理路径和评分过程
5. **多跳推理能力**：支持多层级控股关系查询
6. **真实财报数据**：使用模拟但真实的公司财务信息

### 缺点

1. **依赖 Neo4j 服务**：需要单独部署图数据库
2. **LLM 调用开销**：实体和关系提取需要多次 LLM 调用
3. **实体识别限制**：对复杂公司名称的识别可能不准确
4. **Cypher 查询固定**：当前查询逻辑较为固定，不够灵活

## 改进方向

1. **优化查询逻辑**：使用 LLM 自动生成 Cypher 查询
2. **增加缓存机制**：缓存常见查询结果，提高响应速度
3. **支持增量更新**：支持图谱的增量更新，无需重建
4. **增强实体链接**：提高实体识别和链接的准确性
5. **可视化界面**：开发 Web 界面，支持图谱可视化
6. **更多关系类型**：支持投资、合作等更多公司关系类型
